# Product

## Register

product

## Users

个人开发者 / 技术能力较强的个体用户，把这个网盘作为**自部署的私有云**使用：
存照片、视频、工作文档、长期归档资料，偶尔分享给家人或朋友。
使用场景以桌面浏览器为主（长时间多窗口操作），移动端用于偶尔查看或快传。
要做的核心事情是：可信地把自己的文件放进去、随时找回来、不被打扰地管理它们。

## Product Purpose

NetDisk 是一个单人 / 极小团队规模的网盘系统。它存在的理由是：
现有的私有网盘方案要么是企业管理后台（Ant Design Pro 风），要么是国内传统网盘
的消费向 UI（信息嘈杂、广告位、强营销），都不是一个想认真管理自己数字资产的
个人开发者愿意每天打开的工具。

成功的标志是：
- 用户每次打开都觉得"舒服"——节奏稳、不闪烁、操作有反馈
- 上传 / 转码 / 回收等长任务的状态在任何时刻都是清晰的
- 文件浏览、搜索、预览的路径短，鼠标和键盘都顺手
- 部署完之后能长期默默地用下去，不需要它来"打扰"用户

## Brand Personality

**现代、克制、舒适。**

- **现代** —— 不是装饰意义的"潮"，而是当代工具软件的工艺标准：精确的字体层级、
  统一的圆角与间距节奏、合理使用动效而不是堆砌。让用户一眼看出这不是 2015 年的
  网盘 UI。
- **克制** —— 灰阶为主、强调色少量出现且每次出现都有含义。不卖弄渐变，不用大色块
  填充情绪。视觉权重让位给文件本身（缩略图、文件名、文件结构）。
- **舒适** —— 优先解决"用起来累不累"的问题：足够的留白、不刺眼的对比、长任务有
  恰当的反馈、键盘快捷键、错误提示讲人话、动效在 200ms 量级且总能被 reduced-motion
  关掉。

声音上：直接、就事论事、不卖萌、不商务腔。错误信息说"发生了什么 / 该怎么办"，
不只是 "Something went wrong"。

## Anti-references

明确不要看起来像：

- **通用 SaaS 模板** —— 紫蓝渐变 Hero、emoji icon、glassmorphism 卡片堆叠、
  "Big Number / Small Label" 营销组件、千篇一律的 marketing landing 移植到产品里。
  这是这个项目最容易掉进去的陷阱。
- **传统国内网盘的消费向 UI**（百度网盘 / 115 那种）—— 广告位、强营销弹窗、
  红黄 CTA、信息嘈杂。
- **传统企业管理后台**（Ant Design Pro 蓝白）—— 密集表格 + 蓝色按钮 + 灰底卡片，
  没有任何个性，把一个个人项目做成 OA 系统。
- **玩具 / 儿童感** —— 卡通插画、亮黄亮粉、过度圆角、活泼字体。

## Design Principles

围绕"现代、克制、舒适"展开的可执行原则：

1. **文件优先，UI 退后。** 缩略图、文件名、文件结构是主角，chrome（导航、工具栏、
   状态栏）必须为它们让位。任何会与文件抢视觉权重的装饰元素都要让步。
2. **强调色按事件出现，不按位置出现。** 强调色（primary / accent）只在有用户事件
   需要它的地方出现——选中、进行中、可点击的主操作。不预先在 navbar、卡片边、标题
   下面铺色。避免大面积色块。
3. **长任务必须可被信任。** 上传、转码、回收清理这些长任务在任何时刻都要有
   "在哪一步 / 还要多久 / 失败可重试"的清晰状态。不能让用户怀疑"它是不是卡住了"。
4. **键盘与鼠标双轨。** 文件列表、搜索、预览、上传管理都要能用键盘完成主要操作。
   桌面优先意味着快捷键是认真的一等公民，不是装饰。
5. **动效服务于因果，不服务于氛围。** 动效用来解释"这个东西从哪来 / 到哪去 /
   它在响应你的操作"。禁止纯装饰的 entrance 动画扫过整页。所有动效在
   `prefers-reduced-motion` 下退化为即时切换或交叉淡出。

## Accessibility & Inclusion

目标 **WCAG 2.1 AA**：

- 正文文本对背景 ≥ 4.5:1，大字号文本 ≥ 3:1。占位符同样按正文标准。
- 所有交互元素键盘可达，焦点态可见且对比清晰。
- 所有动效遵守 `prefers-reduced-motion`。
- 表单错误既用颜色又用文案 / 图标双通道传达。
- 国际化目前覆盖中英文（基于 Paraglide）；UI 文案保持简短，方便后续扩展。

没有针对特殊用户群体的额外硬性要求；默认按 AA 标准执行。
